# GameMonitor

Нативное macOS-приложение для **низколатентного** вывода видео и звука с карты захвата на **Apple Studio Display**. Целевой сценарий: **Nintendo Switch 2** → **ezcap Cam Link 4K** → Mac.

## Цепочка сигнала (кто за что отвечает)

```text
Switch 2 (HDMI)  →  Cam Link  →  GameMonitor (Mac)  →  Studio Display
   ↑ главный          ↑ USB/UVC        ↑ только показ
   размер + FPS       масштаб/режим    запрос формата карте
```

| Звено | Что реально настраивает |
|--------|-------------------------|
| **Switch 2** | Разрешение и FPS по HDMI (4K, 1440p, 1080p, 60/50 Hz) |
| **Cam Link** | Что пришло с HDMI → какой поток отдать по USB (часто не совпадает с пресетом в приложении) |
| **GameMonitor** | Какой **UVC-режим попросить** у карты + вывод на экран. **Не меняет Switch** |

Если переключение **4K / 1080p** в приложении **ничего не меняет** — это нормально: картинку уже задал Switch, карта отдаёт то, что может с текущего HDMI. Смотрите в оверлее **«Запрос UVC»** vs **«Карта (UVC)»** — при расхождении будет предупреждение.

## Требования

- macOS 14+
- Apple Silicon Mac
- Nintendo Switch 2 + dock, HDMI 2.1 → Cam Link **HDMI IN**
- Cam Link → **прямой USB-C 3.x** (без хаба)
- Apple Studio Display

## Сборка и запуск

```bash
cd gamemonitor
xcodebuild -project GameMonitor.xcodeproj -scheme GameMonitor -configuration Release -derivedDataPath build build
open build/Build/Products/Release/GameMonitor.app
```

Или откройте `GameMonitor.xcodeproj` в Xcode и запустите (⌘R).

При первом запуске macOS запросит доступ к **камере** и **микрофону** (UVC-захват и аудио с карты).

## Использование

1. Подключите Switch 2 и Cam Link.
2. На Switch 2 выберите вывод (например **1440p @ 60**).
3. В GameMonitor выберите **1440p (2K) · 60 fps** в переключателе «Режим».
4. **Старт** (или **Применить режим**, если захват уже идёт).
5. Картинка на **Studio Display**; **⌘⇧F** — fullscreen; **Esc** — выход.
6. В оверлее проверьте формат, например `2560x1440 … [60–60 fps]`.

## «Запрос UVC» (пресеты в приложении)

Это **не настройки Switch**, а пожелание к карте: «отдай по USB 4K30 / 1080p60 / …». Сработает только если:

1. На Switch уже выбран **совместимый** HDMI-режим.
2. Cam Link **умеет** отдать такой UVC-режим с текущего входа.

| Пресет | Смысл |
|--------|--------|
| **4K · 60 fps** | Попросить 4K60 по USB (нужен 4K@60 на Switch) |
| **4K · 30 fps** | Попросить 4K30 по USB |
| **1440p · 60/30** | Попросить 2K — **сначала 1440p на Switch** |
| **1080p · 120/60** | Попросить Full HD — **сначала 1080p на Switch** |

**Сначала Switch → потом (опционально) пресет в GameMonitor → Стоп/Старт.**

### Switch в режиме «Авто» для TV

В **«Авто»** консоль сама выбирает формат под телевизор/дисплей. Тогда:

- переключение 4K / 1080p **в GameMonitor не изменит картинку** — HDMI уже зафиксирован;
- карта отдаёт **один и тот же UVC**, пока не смените вывод на Switch.

**Что делать:** на Switch отключите «Авто», выберите вручную **1440p @ 60** (или нужный режим). Пресет в приложении — только если карта поддерживает другой UVC с этого входа.

### Почему Switch 4K@60, а на экране ~25 fps?

Две разные причины — смотрите в оверлее **FPS UVC (карта)** и **FPS экран**:

1. **UVC ≈25, экран ≈25** — карта/консоль в **50 Hz (PAL)** или режим 25 fps. На Switch выберите **60 Hz**, пресет **60 fps**.
2. **UVC ≈60, экран ≈25** — с карты идёт 60 fps, но **апскейл 4K→5K (Lanczos)** не успевает на GPU. Выключите апскейл или выберите **«Быстрый»** в настройках; для максимальной плавности оставьте нативный 4K без upscale.

## Плавность кадров

В **Настройки → Дисплей** можно включить **«Плавность кадров»** (по умолчанию выкл.).

- Показывает **последний кадр с карты** на частоте экрана (~60 Hz), без перекодирования YUV (нет «зелёного экрана»).
- При 25 fps с карты ритм на дисплее ровнее, но **новых кадров не появляется** — это не MEMC телевизора.
- Для **игр** лучше выключить и добиться **60 fps с карты** (60 Hz на Switch).

## Апскейл до 5K (Studio Display)

В **Настройки → Дисплей** включите **«Апскейл до разрешения экрана (5K)»** (по умолчанию включено).

- Берёт UVC-кадр и масштабирует GPU до нативного разрешения экрана (Studio Display ≈ **5120×2880**).
- **Lanczos** — чётче; при **4K@60** часто даёт ~25 fps на экране. **Быстрый** — проще фильтр, ближе к 60 fps.
- Это **не** повышает детализацию Switch — только интерполяция для заполнения 5K-экрана.

## Настройки

- **Апскейл до 5K** / режим видео UVC
- Устройство захвата (Cam Link / ezcap)
- Целевой экран
- Fullscreen при запуске
- Громкость / mute
- Статистика (FPS, dropped frames)

## Латентность

Узкие места: карта + USB (1–2 кадра), вывод на 60 Hz дисплей (до 1 кадра). Приложение:

- отбрасывает опоздавшие кадры (`alwaysDiscardsLateVideoFrames`);
- показывает кадры с `DisplayImmediately`;
- не использует `AVCaptureVideoPreviewLayer`;
- не перекодирует видео на Mac, если UVC отдаёт NV12/I420.

### Калибровка

Сфотографируйте таймер на экране Switch и сравните с картинкой на Mac — так оценивается субъективная задержка.

## Upgrade path для 4K @ 60

Для **4K60 end-to-end** с Mac рассмотрите карту уровня AVerMedia GC553G2 и обновление GameMonitor под 2160p60 UVC.

## Устранение неполадок

- **Чёрный экран:** HDMI в **IN** Cam Link; Switch в dock, TV mode; кабель HDMI 2.1.
- **Не 4K30 в оверлее:** откройте «Настройки», проверьте устройство; в Console — лог `[FormatSelector]`.
- **Дропы FPS:** USB напрямую в Mac; закройте OBS; проверьте `Dropped` в оверлее.
- **Нет звука:** разрешите микрофон; в macOS Sound проверьте устройство захвата; громкость в GameMonitor.
- **HDR:** v1 — SDR через UVC; HDR с Switch 2 может не проходить через Cam Link.
